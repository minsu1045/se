# 과제1: 품질 요소 측정 - 클라우드 파일 공유 시스템 (미니 드라이브)

## 1. 대상 시스템 개요

본 문서는 Customer Requirement Document(v0.1)을 기반으로  
클라우드 파일 공유 시스템(미니 드라이브)의 소프트웨어 품질 요소를 측정·분석한다.

해당 시스템은 조직 내 약 200명의 직원이 사용하는 파일 저장, 공유, 검색 기능을 포함한  
클라우드 기반 협업 플랫폼이다.

---

## 2. 품질 요소 측정

### 2.1 Product Operation (제품 운영)

#### 정확성 (Correctness)
- **정의**: 요구사항 명세에 따라 기능이 정확하게 수행되는 정도
- **근거**: 파일 업로드/다운로드, 폴더 생성/삭제/이동, 파일 공유 권한(보기/수정/댓글) 등 기능이 구체적으로 명세됨
- **평가**: 기능 요구사항이 상세히 기술되어 정확성 측정 가능. 단, 각 기능의 성공/실패 기준이 명시되지 않아 보완 필요

#### 신뢰성 (Reliability)
- **정의**: 오류 없이 안정적으로 동작하는 정도
- **근거**: "약 200명이 동시에 사용해도 안정적으로 동작해야 함" (10장 성능 요구사항)
- **평가**: 동시 접속자 수는 명시되어 있으나, 허용 오류율이나 장애 복구 시간(MTTR) 등 수치 기준이 없어 측정이 어려움

#### 효율성 (Efficiency)
- **정의**: 최소한의 자원으로 기능을 수행하는 정도
- **근거**: "파일 검색과 다운로드는 빠르게 동작해야 함" (10장)
- **평가**: 빠른 응답을 요구하고 있으나 구체적인 응답 시간(ms 단위 등) 기준이 없어 정량적 측정 불가

#### 무결성 (Integrity)
- **정의**: 허가되지 않은 접근을 방지하는 정도
- **근거**: 로그인 인증, 권한 기반 접근 제어, 데이터 전송 시 암호화 명시 (9장)
- **평가**: 보안 요구사항이 존재하나, 암호화 방식(TLS 버전 등) 및 인증 방식 세부 기준 부재

#### 사용성 (Usability)
- **정의**: 사용자가 쉽게 사용할 수 있는 정도
- **근거**: "직관적이고 간단한 업로드 방식", "웹 브라우저를 통해 접근, 별도 설치 불필요" (2장, 3장)
- **평가**: 사용 편의성에 대한 언급은 있으나 정량적 사용성 기준(학습 시간, 오류 빈도 등) 미명세

---

### 2.2 Product Revision (제품 개선)

#### 유지보수성 (Maintainability)
- **정의**: 결함 수정 및 기능 변경이 용이한 정도
- **근거**: 요구사항 문서 내 별도 언급 없음
- **평가**: 개발자 관점의 유지보수 기준이 전혀 명세되지 않아 보완 필요
- **예시**: 파일 공유 권한 로직 변경 시 관련 모듈만 수정 가능하도록 모듈화 기준 명세 필요

#### 테스트가능성 (Testability)
- **정의**: 시스템을 테스트하기 용이한 정도
- **근거**: 요구사항 문서 내 별도 언급 없음
- **평가**: 테스트 기준이나 테스트 환경에 대한 내용이 없어 품질 검증이 어려움
- **예시**: 파일 업로드 성공/실패 여부, 권한 없는 사용자 접근 차단 여부 등 테스트 케이스 명세 필요

#### 유연성 (Flexibility)
- **정의**: 기능 변경 및 확장이 용이한 정도
- **근거**: "문서 공동 편집, AI 기반 검색 등 향후 확장 가능성 고려" (11장)
- **평가**: 확장 가능성에 대한 언급은 있으나 구체적인 설계 원칙(모듈화 등)이 명시되지 않음

---

### 2.3 Product Transition (제품 전환)

#### 이식성 (Portability)
- **정의**: 다른 환경으로 이전이 용이한 정도
- **근거**: "웹 브라우저로 접근 가능, 별도 프로그램 설치 불필요" (2장)
- **평가**: 브라우저 기반 접근으로 이식성이 높은 편이나, 지원 브라우저 종류 등 세부 기준 미명세

#### 재사용성 (Reusability)
- **정의**: 구성 요소를 다른 시스템에서 재사용 가능한 정도
- **근거**: 요구사항 문서 내 별도 언급 없음
- **평가**: 재사용성에 대한 명세가 없어 평가 불가

#### 상호운용성 (Interoperability)
- **정의**: 다른 시스템과 연동이 가능한 정도
- **근거**: "링크를 통해 외부 협력 업체와 파일 공유" (5장)
- **평가**: 외부 공유 기능이 일부 언급되어 있으나, API 연동이나 타 시스템 연계 기준은 미명세

---

## 3. 종합 평가

본 요구사항 문서(v0.1)는 기능적 요구사항은 비교적 상세히 기술되어 있으나,  
품질 요소 관점에서 다음과 같은 보완이 필요하다.

| 구분 | 문제점 | 개선 방향 | 기대치 |
|------|--------|-----------|--------|
| 정량적 기준 부재 | 응답 시간, 오류율 등 수치 기준 없음 | 응답 시간 2초 이내,
오류율 1% 미만 등 수치 명세 추가 | 품질 요소를 객관적으로 측정 가능하게 함 |
| 비기능 요구사항 미흡 | 유지보수성, 테스트가능성 누락 | 개발자 관점의 품질 기준 추가 | 개발 및 유지보수 효율 향상 |
| 보안 세부 기준 부족 | 암호화 방식, 인증 방식 미명세 | TLS 버전, 2단계 인증 등 세부 보안 기준 명세 | 보안 취약점 사전 방지 |
| 재사용성/상호운용성 미명세 | API 연동 등 기준 없음 | 외부 시스템 연동 방식 명세 추가 | 향후 시스템 확장 용이 |
