# 요구사항 정의서

# 1. 서론

## 1.1 문서의 목적 및 범위

본 문서는 조직 내 협업을 위한 클라우드 파일 공유 시스템인 **Mini Drive**의 요구사항을 정의하기 위한 문서이다.

본 문서는 Mini Drive가 제공해야 하는 기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항, 기타 요구사항을 정리한다. 이를 통해 시스템 개발 범위를 명확히 하고, 이후 설계 및 구현 단계에서 기준으로 활용한다.

---

## 1.2 대상 시스템 개요

### 1.2.1 프로젝트 정의

Mini Drive는 조직 내부 사용자를 위한 클라우드 기반 파일 관리 및 공유 시스템이다.

현재 조직 내 파일은 이메일, 메신저, 개인 클라우드 서비스 등을 통해 분산되어 공유되는 경우가 많다. 이로 인해 최신 파일 확인, 파일 검색, 외부 공유 시 보안 관리에 어려움이 발생할 수 있다.

Mini Drive는 파일을 중앙에서 관리하고, 업로드, 다운로드, 폴더 관리, 파일 공유, 검색, 버전 관리, 사용자 계정 관리, 보안 관리 기능을 제공하여 조직 내 협업을 지원하는 것을 목표로 한다.

---

### 1.2.2 주요 기능 설명

| ID | 기능명 | 설명 |
|---|---|---|
| F01 | 파일 업로드/다운로드 | 사용자가 업무 파일을 업로드하고 다시 다운로드할 수 있다. |
| F02 | 폴더 관리 | 폴더 생성, 이름 변경, 삭제, 파일 이동을 통해 파일을 정리할 수 있다. |
| F03 | 파일 공유 | 특정 사용자에게 파일을 공유하고 보기, 수정, 댓글 권한을 설정할 수 있다. |
| F04 | 외부 링크 공유 | 외부 협력 업체와 링크를 통해 파일을 공유할 수 있다. |
| F05 | 파일 검색 | 파일 이름, 날짜, 사용자, 파일 유형 등을 기준으로 검색할 수 있다. |
| F06 | 파일 버전 관리 | 이전 버전을 확인, 다운로드, 복원할 수 있다. |
| F07 | 사용자 계정 관리 | 로그인 및 관리자 계정 관리를 지원한다. |
| F08 | 보안 관리 | 권한 기반 접근 제어와 공유 링크 만료 기간 설정을 고려한다. |
